# Russian Book to Audio

Локальный сервис для преобразования текстовых книг на русском языке в аудиокниги с разбиением по главам и частям.

## Возможности

- **Web-интерфейс**: Загрузка `.txt` файлов через удобный браузерный интерфейс.
- **Умная обработка текста**:
  - Нормализация чисел (перевод цифр в слова с помощью `num2words`).
  - Автоматическая расстановка ударений и буквы "ё" для правильного произношения (с помощью `ruaccent`).
  - Интеллектуальное разбиение на предложения (`razdel`) и чанки для стабильной работы нейросетей.
- **Поддержка нескольких TTS-движков**:
  - **XTTS v2 (Docker API)**: Студийное качество и возможность клонирования голоса (Zero-shot).
  - **HuggingFace VITS**: Локальная быстрая модель (`utrobinmv/tts_ru_free_hf_vits_high_multispeaker`).
  - **Coqui TTS**: Встроенная поддержка `xtts_v2`.
  - **Windows SAPI**: Fallback-вариант через встроенный системный синтезатор.
- **Оптимизация аудио**: Автоматическая конвертация сгенерированных WAV-файлов в **MP3** (128 kbps) с помощью `lameenc` для экономии места.
- **Очередь задач**: Фоновая обработка книг с сохранением статуса и прогресса в SQLite.
- **Надежность**:
  - Автоматические повторные попытки (retries) при сбоях генерации (`tenacity`).
  - Контроль оперативной и видеопамяти с автоматическим уменьшением размера чанка.
  - Подробное логирование всех этапов (`loguru`).

## Быстрый старт

### 1. Установка зависимостей

Рекомендуется использовать Python 3.11 или 3.12 (для совместимости со всеми нейросетевыми библиотеками).

```bash
# Базовые зависимости (API, обработка текста, MP3-энкодер)
pip install -r requirements.txt

# Опционально: зависимости для локального GPU-синтеза (VITS / Coqui)
pip install -r requirements-gpu-py311.txt
```

### 2. Подготовка приложения и скачивание локальной модели

Запустите скрипт настройки, который автоматически создаст нужные папки и скачает локальную VITS-модель (`utrobinmv/tts_ru_free_hf_vits_high_multispeaker`) с Hugging Face:

```bash
python setup.py
```

### 3. Запуск сервера XTTS (Рекомендуется для лучшего качества)

Для наилучшего качества озвучки рекомендуется использовать XTTS v2 через Docker.
1. Убедитесь, что после запуска `setup.py` у вас появилась папка `speakers` в корне проекта.
2. Положите туда файл `default_speaker.wav` (5-10 секунд записи голоса, которым хотите озвучивать).
3. Запустите Docker-контейнер:

> **Важно: Сборка кастомного Docker-образа (рекомендуется)**
> Оригинальный образ `daswer123/xtts-api-server:latest` устарел (выдает предупреждение "THIS IMAGE IS DEPRECATED and is scheduled for DELETION") и содержит старую версию PyTorch, которая не поддерживает новейшие видеокарты (например, RTX 5070 Ti с архитектурой `sm_120`).
> 
> В репозитории есть `Dockerfile.xtts`, который собирает сервер с нуля с самой свежей версией PyTorch (CUDA 12.8) и полностью обходит проблему устаревшего образа.
> 
> ```bash
> # 1. Соберите локальный образ (займет пару минут)
> docker build -t xtts-api-server-custom -f Dockerfile.xtts .
> 
> # 2. Запускайте сервер, используя этот новый образ
> # Для PowerShell (Windows) или Linux/macOS:
> docker run --gpus=all -e CUDA_VISIBLE_DEVICES=0 -p 8020:8020 -v "${PWD}/speakers:/app/speakers" -v xtts_models:/app/models xtts-api-server-custom
> 
> # Для командной строки (CMD) в Windows:
> # docker run --gpus=all -e CUDA_VISIBLE_DEVICES=0 -p 8020:8020 -v "%cd%\speakers:/app/speakers" -v xtts_models:/app/models xtts-api-server-custom
> ```

*Приложение автоматически найдет запущенный контейнер на порту 8020 и будет использовать его.*

### 4. Запуск приложения

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Откройте в браузере:
- **Web UI**: [http://127.0.0.1:8000/](http://127.0.0.1:8000/)
- **API Документация**: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

## Структура выходных файлов

Сгенерированные аудиокниги сохраняются в папку `output/<название_книги>/`:

- `chapter_001_part_001.mp3`
- `chapter_001_part_002.mp3`
- `chapter_002_part_001.mp3`
- `manifest.json` (метаданные о книге, использованном движке и файлах)

## Конфигурация

Все основные настройки находятся в `app/config.py` (используется `pydantic-settings`). Вы можете легко изменить:
- Максимальный размер чанка текста.
- Битрейт MP3.
- URL для подключения к Docker-серверу XTTS.

## Разработка и отладка

В проекте настроены строгая типизация и линтеры:
```bash
# Проверка типов
python -m mypy app/

# Форматирование и линтинг
python -m ruff check .
python -m ruff format .
```

Скрипты для отладки и тестирования отдельных компонентов вынесены в папку `debug_scripts/`.

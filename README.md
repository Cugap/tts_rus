# Russian Book to Audio

Локальный сервис для преобразования текстовых книг на русском языке в аудиокниги с разбиением по главам и частям.

## Возможности

- **Web-интерфейс**: Загрузка `.txt` и `.fb2` файлов через удобный браузерный интерфейс.
- **Поддержка FB2**: Парсинг книг в формате FictionBook 2.0 с разбиением по секциям (главам), извлечением заголовков и текста.
- **Валидация FB2**: Автоматическая проверка загружаемых FB2-файлов перед обработкой — детектирование сканированных книг (только изображения), проблем с кодировкой, пустых глав, низкого содержания кириллицы, слишком длинных абзацев и других проблем, влияющих на качество синтеза.
- **Умная обработка текста**:
  - Нормализация чисел (перевод цифр в слова с помощью `num2words`).
  - Автоматическая расстановка ударений и буквы "ё" для правильного произношения (с помощью `ruaccent`).
  - Интеллектуальное разбиение на предложения (`razdel`) и чанки для стабильной работы нейросетей.
- **Поддержка TTS-движков**:
  - **HuggingFace VITS**: Локальная модель (`utrobinmv/tts_ru_free_hf_vits_high_multispeaker`).
  - **Windows SAPI**: Fallback через встроенный системный синтезатор.
- **Оптимизация аудио**: Автоматическая конвертация сгенерированных WAV-файлов в **MP3** (128 kbps) с помощью `lameenc` для экономии места.
- **Очередь задач**: Фоновая обработка книг с сохранением статуса и прогресса в SQLite.
- **Надежность**:
  - Автоматические повторные попытки (retries) при сбоях генерации (`tenacity`).
  - Контроль оперативной и видеопамяти с автоматическим уменьшением размера чанка.
  - Подробное логирование всех этапов (`loguru`).

## Быстрый старт

### 1. Установка зависимостей

Рекомендуется использовать Python 3.11 или 3.12 (для совместимости со всеми нейросетевыми библиотеками).

```bash
# Базовые зависимости (API, обработка текста, MP3-энкодер)
pip install -r requirements.txt

# Опционально: зависимости для локального GPU-синтеза (VITS)
pip install -r requirements-gpu-py311.txt
```

### 2. Подготовка приложения и скачивание локальной модели

Запустите скрипт настройки, который автоматически создаст нужные папки и скачает локальную VITS-модель (`utrobinmv/tts_ru_free_hf_vits_high_multispeaker`) с Hugging Face:

```bash
python setup.py
```

### 3. Запуск приложения

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Откройте в браузере:
- **Web UI**: [http://127.0.0.1:8000/](http://127.0.0.1:8000/)
- **API Документация**: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

## Структура выходных файлов

Сгенерированные аудиокниги сохраняются в папку `output/<название_книги>/`:

- `chapter_001_part_001.mp3`
- `chapter_001_part_002.mp3`
- `chapter_002_part_001.mp3`
- `manifest.json` (метаданные о книге, использованном движке и файлах)

## API Endpoints

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/` | Web-интерфейс |
| `GET` | `/engines` | Список доступных TTS-движков и спикеров |
| `POST` | `/books/validate` | **Валидация FB2** — проверяет файл перед озвучкой без создания задачи |
| `POST` | `/jobs` | Создать задачу на озвучку (принимает `.txt` и `.fb2`) |
| `GET` | `/jobs` | Список всех задач |
| `GET` | `/jobs/{id}` | Статус конкретной задачи |

### Валидация FB2 (`POST /books/validate`)

Эндпоинт проверяет FB2-файл на пригодность к синтезу речи. Возвращает:
- `ok` — прошёл ли файл проверку (`true`/`false`)
- `issues` — список проблем с кодами и описанием
- `stats` — статистика файла (символы, абзацы, главы, изображения и т.д.)

**Коды проблем валидации:**

| Код | Описание |
|---|---|
| `xml_parse` | Файл повреждён или не является корректным XML |
| `not_fb2` | Отсутствует корневой элемент `FictionBook` |
| `no_body` | Нет основного текста (отсутствует `body`) |
| `no_text` | В FB2 нет текста для озвучки |
| `image_only` | Книга состоит только из изображений (скан) |
| `table_only` | Есть таблицы, но нет текста |
| `too_short` | Слишком мало символов (меньше 50) |
| `empty_chapters` | Все главы пустые |
| `low_letter_ratio` | Мало буквенных символов — возможна проблема с кодировкой |
| `encoding` | Много символов замены (�) — проблема с кодировкой |
| `short_chapters` | Все главы слишком короткие |
| `low_cyrillic_ratio` | Мало кириллицы (< 50%) — текст может быть не на русском |
| `high_non_alpha_ratio` | Много небуквенных символов (> 80%) |
| `long_paragraphs` | Очень длинные абзацы (> 3000 символов) — могут обрезаться |
| `long_chapter_title` | Слишком длинные названия глав (> 120 символов) |

## Конфигурация

Все основные настройки находятся в `app/config.py` (используется `pydantic-settings`). Вы можете легко изменить:
- Максимальный размер чанка текста.
- Битрейт MP3.

## Разработка и отладка

В проекте настроены строгая типизация и линтеры:
```bash
# Проверка типов
python -m mypy app/

# Форматирование и линтинг
python -m ruff check .
python -m ruff format .
```

Конфигурация линтеров и типизации находится в `pyproject.toml`.
